# Chantier 1 — Re-cadrage créatif premium du landing KORYXA Formation

Date: 2026-06-23
Repo concerné: `koryxa_formation`
Statut: cadrage uniquement, aucune modification frontend dans ce chantier

Objectif: reprendre la direction créative du landing public pour obtenir une page beaucoup plus vivante, premium, visuelle, animée et centrée utilisateur.

---

## 1. Diagnostic honnête de la version actuelle

La version actuellement poussée est techniquement propre, mais elle ne répond pas encore au niveau visuel attendu.

Problèmes identifiés:

```txt
- page trop plate;
- trop de cartes statiques;
- pas assez d'images réelles;
- énergie visuelle insuffisante;
- animations trop discrètes;
- hero pas assez immersif;
- design trop documenté / trop cadrage interne;
- vocabulaire parfois trop froid ou technique;
- manque d'effet premium dès le premier écran;
- manque de sensation de marque forte.
```

Conclusion:

```txt
Il faut traiter la prochaine version comme une vraie refonte créative, pas comme une simple amélioration de composants.
```


## 10. Règles performance

Les animations doivent être visibles, mais pas lourdes.

Règles:

```txt
- pas de vidéo hero de 20 MB;
- viser une vidéo courte optimisée ou image + motion CSS;
- webp pour photos;
- mp4/webm optimisé si vidéo;
- lazy loading pour images secondaires;
- framer-motion pour interactions;
- SVG/CSS pour schémas animés;
- Lottie uniquement si licence claire et fichier léger;
- pas de GIF lourd.
```

Objectifs poids:

```txt
Hero video si utilisée: idéalement < 4 MB
Hero image: < 350 KB
Images secondaires: < 250 KB chacune
Lottie: < 100 KB
```

## 11. Règles de licence

Avant intégration d'un asset:

```txt
[ ] source vérifiée
[ ] auteur noté
[ ] licence notée
[ ] URL notée
[ ] pas de logo visible problématique
[ ] pas de marque dominante
[ ] pas d'image premium payante prise par erreur
[ ] asset optimisé
```

Créer ou mettre à jour:

```txt
frontend/public/assets/landing/ASSET-SOURCES.md
```

Format:

```md
| Fichier | Source | Auteur | URL | Licence | Date | Usage |
|---|---|---|---|---|---|---|
```
