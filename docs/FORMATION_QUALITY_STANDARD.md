# KORYXA Formation — Standard qualité mondiale

## 1. Ambition

La formation **Analyse de Données avec Python** doit être traitée comme un produit pédagogique premium, pas comme une suite de textes ou de notebooks. Elle doit pouvoir être suivie par un débutant sans accompagnement constant, tout en donnant une impression de sérieux à un professionnel.

Objectif central :

```text
Former un apprenant débutant à devenir opérationnel en analyse de données avec Python, avec une expérience claire, belle, interactive, exigeante et orientée projets réels.
```

La promesse qualité KORYXA :

```text
Clarté d'un coach personnel.
Rigueur d'une formation professionnelle.
Interactivité d'une plateforme moderne.
Esthétique d'un produit premium.
Utilité d'un parcours orienté métier.
```

---

## 2. Principes pédagogiques non négociables

### 2.1 Objectif mesurable avant contenu

Chaque module doit commencer par des objectifs formulés du point de vue de l'apprenant :

```text
À la fin de ce module, tu sauras faire quoi ?
```

Exemples :

```text
À la fin du module 1, tu sauras créer un projet Python Data propre sur ton ordinateur.
À la fin du module 2, tu sauras manipuler variables, conditions, boucles et fonctions pour résoudre de petits problèmes data.
À la fin du module Pandas, tu sauras charger, explorer, filtrer, nettoyer et résumer un fichier CSV.
```

### 2.2 Progression guidée

Chaque notion doit suivre la séquence :

```text
Voir → Comprendre → Tester → Corriger → Refaire → Valider
```

Pour le code :

```text
Lire → Exécuter → Modifier → Observer → Expliquer → Valider
```

Un apprenant ne doit pas seulement lire. Il doit pratiquer rapidement, obtenir un retour, corriger, puis refaire.

### 2.3 Exemples guidés avant autonomie

Chaque nouvelle notion doit être introduite avec un exemple guidé complet avant de demander un exercice autonome.

Structure obligatoire :

```text
1. Situation concrète
2. Code complet
3. Explication ligne par ligne
4. Résultat attendu
5. Erreur fréquente
6. Mini-modification à faire par l'apprenant
```

### 2.4 Feedback fréquent

Chaque module doit contenir des points de feedback courts :

```text
✅ Tu as réussi si...
⚠️ Vérifie ceci si...
💡 Indice si tu bloques...
❌ Erreur fréquente...
```

Les évaluations doivent être fréquentes, peu intimidantes, et utiles pour progresser.

### 2.5 Charge mentale maîtrisée

Un module débutant ne doit pas présenter trop de concepts à la fois.

Règles :

```text
Une section = une idée principale.
Un exemple = une compétence précise.
Un tableau = une clarification, pas une décoration.
Un exercice = une action mesurable.
```

### 2.6 Apprentissage orienté métier

Les exemples doivent utiliser des situations réelles : ventes, paiements, clients, stocks, campagnes, performances, données propres ou sales.

Exemples préférés :

```text
chiffre d'affaires
ventes par produit
paiements en attente
clients actifs
stock restant
performance d'une campagne
analyse d'un fichier CSV
```

Éviter les exemples trop scolaires sans valeur métier quand une alternative business est possible.

## 3. Structure officielle d'un module KORYXA

Chaque module doit respecter ce template.

```text
1. Hero du module
2. Objectifs mesurables
3. Carte “Pourquoi ce module compte”
4. Pré-requis
5. Plan clair du module
6. Concepts expliqués simplement
7. Exemples guidés
8. Zones “Teste toi-même”
9. Erreurs fréquentes + corrections
10. Tableaux récapitulatifs
11. Exercices progressifs
12. Mini-défi métier
13. Quiz court
14. Checklist de validation
15. Ressources officielles et complémentaires
16. Passage au module suivant
```

### 3.1 Hero du module

Le début du module doit donner envie et rassurer.

Contenu :

```text
Titre fort
Promesse concrète
Durée estimée
Niveau
Résultat final attendu
```

Exemple :

```text
Module 1 — Installer son environnement et créer son premier projet Python Data

Dans ce module, tu vas préparer ton ordinateur pour travailler comme un analyste data : Python, Anaconda, Jupyter Notebook, Google Colab, VS Code, environnement virtuel et premier mini-projet.

Résultat final : tu auras un dossier de projet propre, un notebook fonctionnel et un premier script Python qui calcule un chiffre d'affaires.
```

### 3.2 Objectifs mesurables

Chaque objectif doit utiliser un verbe d'action : installer, créer, exécuter, expliquer, corriger, organiser, analyser, présenter.

Mauvais :

```text
Comprendre Python.
```

Meilleur :

```text
Créer un fichier Python, l'exécuter et expliquer le résultat affiché.
```

### 3.3 Concepts expliqués simplement

Chaque concept doit répondre à cinq questions :

```text
C'est quoi ?
Pourquoi c'est utile ?
Quand l'utiliser ?
Comment l'utiliser ?
Quelles erreurs éviter ?
```

### 3.4 Exemple guidé

Format obligatoire :

```text
Situation : une boutique veut calculer son chiffre d'affaires.
Code : exemple complet.
Résultat attendu : ce qui doit s'afficher.
Explication : ligne par ligne.
À toi de jouer : petite modification.
```

### 3.5 Zone “Teste toi-même”

Chaque notion importante doit proposer une action.

Exemples :

```text
Change le prix et observe le total.
Ajoute une nouvelle vente dans la liste.
Modifie le nom du fichier CSV.
Crée une variable client_actif.
Corrige le code cassé.
```

### 3.6 Erreurs fréquentes

Chaque module doit anticiper les erreurs réalistes.

Format :

```text
Erreur affichée
Cause probable
Solution simple
Comment vérifier que c'est corrigé
```

### 3.7 Mini-défi métier

Chaque module doit finir par une petite production utile.

Exemples :

```text
Module 1 : créer le projet local analyse_ventes.
Module 2 : calculer un total, une moyenne et une décision simple.
Module 3 : organiser des clients dans listes/dictionnaires.
Module 4 : charger un CSV et compter les lignes.
Module Pandas : trouver les 5 meilleurs produits.
Module EDA : formuler 3 conclusions business.
```

## 4. Standard d'écriture professionnelle

Le style doit être clair, professionnel, précis et accessible.

### 4.1 Ton

Le ton doit être :

```text
clair
rassurant
exigeant
professionnel
orienté action
accessible aux débutants
```

À éviter :

```text
langage vague
phrases longues inutiles
ton trop scolaire
humour forcé
jargon sans explication
texte qui ressemble à un vieux manuel
```

### 4.2 Règle de lisibilité

Chaque section doit être courte et structurée.

Préférer :

```text
Une explication courte.
Un exemple.
Une action.
Un résultat attendu.
```

Éviter :

```text
Un long paragraphe théorique sans pratique.
```

### 4.3 Vocabulaire débutant

Tout terme technique doit être expliqué dès la première apparition.

Exemples :

```text
Un environnement virtuel est un espace isolé pour installer les bibliothèques d'un projet sans mélanger tout l'ordinateur.
Un notebook est un cahier interactif où tu peux mélanger texte, code et résultats.
Un terminal est un outil qui permet de donner des commandes à l'ordinateur.
```

### 4.4 Niveau de détail attendu

Quand l'apprenant doit installer ou créer quelque chose, on détaille comme pour quelqu'un qui débute réellement :

```text
où cliquer
quoi écrire
quoi vérifier
ce qui doit apparaître
quoi faire si ça ne marche pas
```

## 5. Standard visuel des cours

Les cours ne doivent pas ressembler à une page brute de notebook. Ils doivent être affichés comme une interface pédagogique premium.

### 5.1 Composants pédagogiques obligatoires

Chaque module doit pouvoir afficher :

```text
Hero de module
Carte objectif
Carte concept
Bloc exemple guidé
Bloc code
Bloc résultat attendu
Bloc “À retenir”
Bloc “Erreur fréquente”
Bloc “À toi de jouer”
Tableau comparatif
Checklist
Quiz
Mini-défi
Ressources
```

### 5.2 Hiérarchie visuelle

Règles :

```text
Un titre principal fort.
Des sous-titres lisibles.
Des cartes aérées.
Des tableaux propres.
Des blocs d'alerte distincts.
Des CTA d'apprentissage visibles.
```

### 5.3 Esthétique KORYXA Formation

Direction :

```text
clair
premium
sobre
moderne
mobile-first
orienté confiance
```

Palette recommandée :

```text
Fond : white, slate-50, emerald-50, blue-50
Texte fort : slate-950
Texte secondaire : slate-600 / slate-700
Accent principal : emerald / teal
Accent secondaire : blue
Premium sombre : slate-950 / blue-950
```

À éviter :

```text
fond sombre partout
trop d'emojis
cartes collées
textes minuscules
trop de couleurs
pages plates
tableaux illisibles sur mobile
```

### 5.4 Mobile-first

Chaque cours doit être confortable sur mobile.

Règles :

```text
Les tableaux doivent scroller proprement ou devenir lisibles.
Les blocs de code doivent scroller horizontalement sans casser la page.
Les boutons doivent être faciles à toucher.
Les sections doivent respirer.
Les longs textes doivent être découpés.
```

## 6. Standard interactif

L'interactivité est une exigence produit.

### 6.1 Niveau 1 — Exécution libre sous exemple

Sous les exemples importants, ajouter :

```text
éditeur de code
bouton Exécuter
sortie affichée
bouton Réinitialiser
message d'erreur lisible
```

### 6.2 Niveau 2 — Exercices vérifiés

Pour les exercices simples, ajouter une vérification :

```text
✅ Correct
❌ Pas encore
💡 Indice
🔁 Réessaie
```

### 6.3 Niveau 3 — Mini-labs data

Pour les modules avancés :

```text
dataset préchargé
code modifiable
résultat tableau
graphique
question métier
validation finale
```

### 6.4 Choix technique recommandé

Pour la première version interactive, privilégier une exécution Python côté navigateur avec Pyodide.

Raison :

```text
plus sûr que d'exécuter du code apprenant sur notre serveur
plus simple pour un premier périmètre maîtrisé
adapté aux exercices Python de base et data
compatible avec une expérience intégrée dans le cours
```

## 8. Standard spécifique du Module 1

### Nouveau titre

```text
Module 1 — Installer son environnement et créer son premier projet Python Data
```

### Rôle

Le Module 1 doit rendre l'apprenant autonome techniquement.

Objectif :

```text
Même sans expérience, l'apprenant doit pouvoir installer les outils, créer un projet, ouvrir un notebook, exécuter du code et comprendre les erreurs d'installation fréquentes.
```

### Contenu obligatoire

```text
Python : c'est quoi et pourquoi on l'utilise
Anaconda : rôle, installation, usage débutant
Jupyter Notebook : cahier interactif, cellules, exécution
Google Colab : alternative sans installation locale
VS Code : éditeur professionnel
Terminal / invite de commandes : commandes de base
Environnement virtuel venv : pourquoi, création, activation
Création d'un dossier projet propre
Différence entre .py et .ipynb
Installation des bibliothèques pandas, numpy, matplotlib, jupyter
Premier projet analyse_ventes
Erreurs fréquentes et corrections
Checklist finale
```

### Structure de projet attendue

```text
KORYXA_FORMATION/
├── notebooks/
│   ├── module_1_installation.ipynb
│   └── module_2_bases_python.ipynb
├── datasets/
│   └── ventes_exemple.csv
├── projets/
│   └── analyse_ventes/
│       ├── main.py
│       ├── notebook.ipynb
│       └── ventes.csv
└── notes.txt
```

### Exemple de première victoire

```python
ventes = [12000, 15000, 29000, 8000]
total = sum(ventes)
print("Chiffre d'affaires :", total, "FCFA")
```

Résultat attendu :

```text
Chiffre d'affaires : 64000 FCFA
```
